# Flow Ship

1. review 完了と test evidence を確認する
2. `SUMMARY.md` に outcome、delivered、remaining work、evidence を書く
3. PR か main 直 merge かを整理する
4. 未解決事項があれば隠さず残す
5. `REVIEW.md` が `Ship ready: yes` でない場合は出荷扱いにしない
